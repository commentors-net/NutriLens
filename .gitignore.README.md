# Git Ignore Configuration

This `.gitignore` file is configured to exclude:

## Development Files
- IDE settings (VS Code, IntelliJ, etc.)
- OS-specific files (macOS .DS_Store, Windows Thumbs.db)
- Temporary files and caches

## Flutter/Dart
- `.dart_tool/`, `.flutter-plugins`, `.packages`
- Generated plugin registrants
- Build artifacts and intermediates
- Lock files (pubspec.lock should be committed for apps, ignored for packages)

## Android
- `local.properties` (Flutter SDK path, must be generated locally)
- `.gradle/` and build directories
- IDE files (`*.iml`, `.idea/`)
- **Keystores and signing files** (`*.keystore`, `*.jks`, `key.properties`)
- APK/AAB artifacts (unless intentionally tracked)

## iOS
- `Pods/` (CocoaPods dependencies)
- Xcode user settings (`xcuserdata/`)
- Derived data and build intermediates
- Framework files generated by Flutter
- Provisioning profiles (`.mobileprovision`)

## Backend (Python)
- Virtual environments (`venv/`, `.venv/`)
- Python bytecode (`__pycache__/`, `*.pyc`)
- Test/coverage artifacts (`.pytest_cache/`, `.coverage`)
- Local databases (`*.db`, `*.sqlite`)
- **Environment variables** (`.env` — use `.env.example` as template)

## Secrets & Sensitive Data
- API keys and credentials
- Certificate files (`.pem`, `.p12`, `.crt`)
- Firebase configuration files
- Service account JSON files
- Signing keystores and certificates

## Media & Build Outputs
- Images (`*.jpg`, `*.png`, `*.gif`)
- Compiled binaries (`*.apk`, `*.aab`, `*.ipa`)
- Archives and compressed files

---

## What SHOULD be committed:
✅ Source code (`.dart`, `.py`, `.kt`, `.swift`)
✅ Configuration templates (`.env.example`, `key.properties.example`)
✅ Build configuration (Gradle files, Podfile, pubspec.yaml)
✅ JSON schemas (`shared/schemas/*.json`)
✅ Documentation (`.md` files)
✅ Tests

## What should NOT be committed:
❌ Local environment configuration (`.env`, `local.properties`)
❌ IDE-specific settings
❌ Build artifacts and intermediates
❌ Secrets, keys, and certificates
❌ Large media files (unless essential assets)
❌ Database files with real data

---

**Before committing sensitive files, always check `.gitignore` and use `git status` to verify what will be tracked.**
